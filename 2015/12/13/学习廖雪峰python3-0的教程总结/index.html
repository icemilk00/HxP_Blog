学习廖雪峰python3.0的教程总结

　　之前一只想学python，苦于意志力不够坚强，拖了很久，最近终于花了大概一个月时间，把廖雪峰的python3.0教程学完了。写篇总结。
　　每天的学习时间都是下班回家之后，大概平均2-3小时左右，周六日也一样，是的，学习时间并没有是周末就加长，不要问我为什么，懒癌不需要解释。总共花了大概1个月左右。

　　廖大的教程分为两部分，一部分是python语法教程，一部分是实践项目。

　　语法教程,前三分之二都比较容易，基础语法，有些别的语言的底子的话，甚至可以不用跟着敲实例代码，到了高级编程阶段，就需要一些理解了，我是被卡在协程那里，花了一些时间来理解。高级编程那块最好是把代码跟着敲了，练手+理解。有些东西实在理解不了就跳过，后面有用到的时候回来查，会更有效一些。
　　实践教程,实践教程确实很棒，做到最后会得到一个完整版的blog网站。而且廖大的教程里自己写了orm模块和web框架，而没有用第三方库，坏处是我们在学习的时候需要花更多的时间理解这块代码，而好处是对orm和web框架结构有了更深层次的了解，在之后用一些第三方的框架的时候，不会只浮于表面的用法，而不知其工作原理。廖大这块的实践代码选择自己写还是非常妥当的处理。也学到了很多东西。

我把示例代码做了详细的注释和理解，放到了自己的github上,如果能帮到别的学习者，也算缘分了哈哈。

遇到的一些问题

1.在Day10的教程中，会遇到提交注册表单之后服务器没反应的状况:

由于你用的uikit版本和教程里的uikit版本不一样导致static目录下的文件不一样，你在提交表单时，代码里要加载的js文件，其实你的static目录里根本没有，导致加载不上，而运行失败。
解决办法就是把教程github上的源代码static目录覆盖掉你的static，保证你的static目录和教程的static目录里文件一样。

2.在Day11的教程中，会遇到创建博客日志需要登录，登录完后创建仍需登录的问题：

由于是需要管理员才能创建日志的，所以你需要新注册一个管理员用户，可以在代码中写判断，如果是管理员的邮箱的话，在插入user表的时候，把admin字段设为YES就ok了。或者直接手动在数据库中添加条目，并把admin字段设为YES

3.实践项目中，最后会遇到首页，点击下一页，但是无响应的问题，就是跳转不了页数：

需要在handlers.py中的 index(*, page=’1’) 函数中的 page = Page(num) 改成 page = Page(num, page_index)

#首页，会显示博客列表
@get('/')
def index(*, page='1'):
    #获取到要展示的博客页数是第几页
    page_index = get_page_index(page)
    #查找博客表里的条目数
    num = yield from Blog.findNumber('count(id)')
    #通过Page类来计算当前页的相关信息
    page = Page(num, page_index)
    #如果表里没有条目，则不需要系那是
    if num == 0:
        blogs = []
    else:
        #否则，根据计算出来的offset(取的初始条目index)和limit(取的条数)，来取出条目
        blogs = yield from Blog.findAll(orderBy='created_at desc', limit=(page.offset, page.limit))
        #返回给浏览器
    return {
        '__template__': 'blogs.html',
        'page': page,
        'blogs': blogs
    }

总结

廖老师的教程是很棒的一篇教程，尤其在实践部分的代码，是有很干得代码，包括封装思路，模块解耦，代码设计等，而且实践里的orm模块和web框架都是自己写的，能让初学者学到除了python语法之外的很多东西。十分推荐。
