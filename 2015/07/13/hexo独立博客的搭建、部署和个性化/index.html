hexo独立博客的搭建、部署和个性化

  • hexo
  • 博客搭建

身为一只混迹在高逼格IT圈的程序员，一个个性化的博客还是必须的。因为需要不断的学习和积累，之前用的csdn的博客，后来想搭建一个属于自己的博客，机缘巧合正好看到hexo，就怼了进来，粗看了下觉得还不错，就找教程搭了一下，最后发现还算简单易用，所以稍微记录一下。

这个博客也是为自己积累一些东西用，并且提示自己不断学习，且不要浮躁。

1. Hexo介绍

hexo是出自一个台湾同胞tommy351之手，是一个静态Node.js的博客程序，我们可以把hexo生成的网页放到Github之类的平台上，也可以放到我们自己的服务器上，反正你高兴就好。

2. 环境部署

我们这里主要用一些主流的部署，就是Hexo生成的blog，部署到GitHub Pages上, 然后域名绑定，再然后设置主题，网站统计用百度统计，评论用多说评论，分享用jiaThis，之后再有什么加的，我在补充。

3. 环境准备

安装Node
Node.js的博客，当然要先装Node,到Node官方网站，下载最新版本,然后无脑安装。

安装Git
应该大多数猿类都有Git吧，我就默认你有了，没有的话，Git官方文档，进去左边可以调语言，配好Git。

安装Sublime
其实任意一款你熟悉且喜欢的文本编辑工具就行，我这里用Sublime，因为我电脑上正好有这玩意，并且它正好支持MarkDown，而已。下载就自行度娘吧，我的貌似还是没破解版。

GitHub
这也应该是程序猿标配吧，所以这里我也不多细说，没有的话，去GitHub注册个号，然后做一下配置。主要是ssh的配置和global配置，很容易。
看这两个东西：
配置SSH-Keys，
配置SSH出现Error: Permission denied解决办法

4. 本地配置Hexo

安装hexo
安装好Node.js和Git后就可以开始搞hexo了，先安装：

npm install -g hexo

初始化hexo
cd到你想放blog的目录，执行：

hexo init

生成静态页面
cd到你的blog目录，执行：

hexo generate

然后会发现你的hexo/public/ 目录下多了一堆文件夹。

本地预览
如果上面一切顺利，那么你在命令行输入：

hexo server

然后去浏览器访问 127.0.0.1:4000就可以看到你的hexo博客的样子了，并且你在做一些配置或者别的修改时，只要刷一下网页，就可以实施看到你后台做的变动在前端的显示了。

5. 部署Hexo到你的GitHub

现在你的博客已经初步成型，是不是灰常简单，那么我们现在要把它发布到网上，这里我们选择使用GitHub免费提供的GitHub Pages。

新建仓库

点击Create repository，这样你即将存放hexo博客代码的仓库就创建好了。

配置
找到你hexo博客目录下的全局配置文件：_config.yml, 用编辑工具（或vim）打开，找到最下边deploy部分，设置GitHub地址。

deploy:
  type: git
  repo: https://github.com/your_GitHub_account/your_blog_name.git
  branch: gh-pages

上面的repo就是你刚才新建的仓库的地址,譬如我的就是：https://github.com/icemilk00/hxp_blog.git，分支填写gh-pages分支，这个分支下，GitHub会自动给你生成可访问的站点。

这里注意，标签冒号后边一定要加一个空格，否则都不生效。

上传
然后直接执行命令：

hexo deploy

出现deploy的打印信息，就说明正在上传。不出意外会上传成功。

如果你发现你执行了这条命令后，没有错误提示也没有成功提示，即无任何终端输出的话，请回_config.yml里查看刚才的deploy配置是不是没有在标签冒号后加空格。

上传成功后，打开你的GitHub，就会看到如下的图,然后选择gh-pages分支:

你可以看到hexo博客文件，然后找到右侧setting选项，并点进去：

你就可以在GitHub Pages 里看到你hexo博客的地址了：

这样，属于你自己的博客就算正式发布上线了，接着我们要去绑定一个域名。
