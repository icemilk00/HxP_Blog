Git常用命令使用说明

git

为了复习并记录下常用的Git命令

从来没有接触过git的同学可以参考这个:
1.我学Git就是参考的这个
2.官方文档

初始化一个Git仓库：

git init

添加文件到仓库：

git add <file>

git commit

查看工作区的Git状态：

git status

查看修改：

git diff

查看提交历史：

git log

查看命令历史：

git reflog

回滚到某个版本：

git reset --hard commit_id

commit_id 可以通过查看提交历史，来获取想要回滚的版本id

丢弃工作区的修改：

git checkout -- file

会同步到最近一次 git add 的内容

丢弃暂存区内容：

git reset HEAD file

会把最近一次 git add 的内容干掉，然后再执行上面的 git checkout — file，可以把工作区和暂存区同步

丢弃版本库的内容：

请参考上面的 回滚到某个版本 的内容。

删除版本库某个文件：

git rm

关联一个远程库：

git remote add origin git@server-name:path/repo-name.git

第一次推送master分支所有内容：

git push -u origin master

之后的修改推送：

git push origin master

从远程库克隆一个本地库：

git clone git@server-name:path/repo-name.git

查看分支：

git branch

创建分支：

git branch <name>

切换分支：

git checkout <name>

创建 + 切换分支：

git checkout -吧 <name>

合并某分支到当前分支：

git merge <name>

删除分支：

git branch -d <name>

查看分支合并图：

git log --graph

保存当前的工作现场：

git stash

恢复保存的工作现场到工作区：

git stash apply

删除保存的工作现场：

git stash drop

恢复工作现场到工作区，并删除保存的现场：

git stash pop

强行删除某个分支：

git brach -D <name>

查看远程库信息：

git remote

查看远程库详细信息：

git remote -v

推送分支：

git push origin <branch_name>

抓取远程新提交：

git pull

在本地创建和远程分支对应的分支：

git checkout -b branch_name origin/branch_name

简历本地分支和远程分支的关联：

git branch --set-upstream branch_name origin/branch_name

打标签：

git tag <tag_name>

查看所有标签：

git tag

对某个版本打标签：

git tag <tag_name> commit_id

查看标签信息：

git show <tag_name>

创建带有说明的标签：

git tag -a <tag_name> -m "tag_desprition" commit_id

用私钥签名打一个标签

git tag -a <tag_name> -m "tag_desprition" commit_id

删除标签：

git tag -d <tag_name>

推送某个标签到远程

git push origin <tag_name>

一次性推送全部尚未推送到远程的本地标签：

git push origin --tags

从远程库删除标签：

git push origin :refs/tags/<tag_name>
