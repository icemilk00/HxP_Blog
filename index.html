Git常用命令使用说明

为了复习并记录下常用的Git命令

从来没有接触过git的同学可以参考这个:
1.我学Git就是参考的这个
2.官方文档

初始化一个Git仓库：

git init

添加文件到仓库：

git add <file>

git commit

查看工作区的Git状态：

git status

查看修改：

git diff

查看提交历史：

git log

查看命令历史：

git reflog

回滚到某个版本：

git reset --hard commit_id

commit_id 可以通过查看提交历史，来获取想要回滚的版本id

丢弃工作区的修改：

git checkout -- file

会同步到最近一次 git add 的内容

丢弃暂存区内容：

git reset HEAD file

会把最近一次 git add 的内容干掉，然后再执行上面的 git checkout — file，可以把工作区和暂存区同步

丢弃版本库的内容：

请参考上面的 回滚到某个版本 的内容。

删除版本库某个文件：

git rm

关联一个远程库：

git remote add origin git@server-name:path/repo-name.git

第一次推送master分支所有内容：

git push -u origin master

之后的修改推送：

git push origin master

从远程库克隆一个本地库：

git clone git@server-name:path/repo-name.git

查看分支：

git branch

创建分支：

git branch <name>

切换分支：

git checkout <name>

创建 + 切换分支：

git checkout -吧 <name>

合并某分支到当前分支：

git merge <name>

删除分支：

git branch -d <name>

查看分支合并图：

git log --graph

保存当前的工作现场：

git stash

恢复保存的工作现场到工作区：

git stash apply

删除保存的工作现场：

git stash drop

恢复工作现场到工作区，并删除保存的现场：

git stash pop

强行删除某个分支：

git brach -D <name>

查看远程库信息：

git remote

查看远程库详细信息：

git remote -v

推送分支：

git push origin <branch_name>

抓取远程新提交：

git pull

在本地创建和远程分支对应的分支：

git checkout -b branch_name origin/branch_name

简历本地分支和远程分支的关联：

git branch --set-upstream branch_name origin/branch_name

打标签：

git tag <tag_name>

查看所有标签：

git tag

对某个版本打标签：

git tag <tag_name> commit_id

查看标签信息：

git show <tag_name>

创建带有说明的标签：

git tag -a <tag_name> -m "tag_desprition" commit_id

用私钥签名打一个标签

git tag -a <tag_name> -m "tag_desprition" commit_id

删除标签：

git tag -d <tag_name>

推送某个标签到远程

git push origin <tag_name>

一次性推送全部尚未推送到远程的本地标签：

git push origin --tags

从远程库删除标签：

git push origin :refs/tags/<tag_name>

hexo独立博客的搭建、部署和个性化

身为一只混迹在高逼格IT圈的程序员，一个个性化的博客还是必须的。因为需要不断的学习和积累，之前用的csdn的博客，后来想搭建一个属于自己的博客，机缘巧合正好看到hexo，就怼了进来，粗看了下觉得还不错，就找教程搭了一下，最后发现还算简单易用，所以稍微记录一下。

这个博客也是为自己积累一些东西用，并且提示自己不断学习，且不要浮躁。

1. Hexo介绍

hexo是出自一个台湾同胞tommy351之手，是一个静态Node.js的博客程序，我们可以把hexo生成的网页放到Github之类的平台上，也可以放到我们自己的服务器上，反正你高兴就好。

2. 环境部署

我们这里主要用一些主流的部署，就是Hexo生成的blog，部署到GitHub Pages上, 然后域名绑定，再然后设置主题，网站统计用百度统计，评论用多说评论，分享用jiaThis，之后再有什么加的，我在补充。

3. 环境准备

安装Node
Node.js的博客，当然要先装Node,到Node官方网站，下载最新版本,然后无脑安装。

安装Git
应该大多数猿类都有Git吧，我就默认你有了，没有的话，Git官方文档，进去左边可以调语言，配好Git。

安装Sublime
其实任意一款你熟悉且喜欢的文本编辑工具就行，我这里用Sublime，因为我电脑上正好有这玩意，并且它正好支持MarkDown，而已。下载就自行度娘吧，我的貌似还是没破解版。

GitHub
这也应该是程序猿标配吧，所以这里我也不多细说，没有的话，去GitHub注册个号，然后做一下配置。主要是ssh的配置和global配置，很容易。
看这两个东西：
配置SSH-Keys，
配置SSH出现Error: Permission denied解决办法

4. 本地配置Hexo

安装hexo
安装好Node.js和Git后就可以开始搞hexo了，先安装：

npm install -g hexo

初始化hexo
cd到你想放blog的目录，执行：

hexo init

生成静态页面
cd到你的blog目录，执行：

hexo generate

然后会发现你的hexo/public/ 目录下多了一堆文件夹。

本地预览
如果上面一切顺利，那么你在命令行输入：

hexo server

然后去浏览器访问 127.0.0.1:4000就可以看到你的hexo博客的样子了，并且你在做一些配置或者别的修改时，只要刷一下网页，就可以实施看到你后台做的变动在前端的显示了。

5. 部署Hexo到你的GitHub

现在你的博客已经初步成型，是不是灰常简单，那么我们现在要把它发布到网上，这里我们选择使用GitHub免费提供的GitHub Pages。

新建仓库

点击Create repository，这样你即将存放hexo博客代码的仓库就创建好了。

配置
找到你hexo博客目录下的全局配置文件：_config.yml, 用编辑工具（或vim）打开，找到最下边deploy部分，设置GitHub地址。

deploy:
  type: git
  repo: https://github.com/your_GitHub_account/your_blog_name.git
  branch: gh-pages

上面的repo就是你刚才新建的仓库的地址,譬如我的就是：https://github.com/icemilk00/hxp_blog.git，分支填写gh-pages分支，这个分支下，GitHub会自动给你生成可访问的站点。

这里注意，标签冒号后边一定要加一个空格，否则都不生效。

上传
然后直接执行命令：

hexo deploy

出现deploy的打印信息，就说明正在上传。不出意外会上传成功。

如果你发现你执行了这条命令后，没有错误提示也没有成功提示，即无任何终端输出的话，请回_config.yml里查看刚才的deploy配置是不是没有在标签冒号后加空格。

上传成功后，打开你的GitHub，就会看到如下的图,然后选择gh-pages分支:

你可以看到hexo博客文件，然后找到右侧setting选项，并点进去：

你就可以在GitHub Pages 里看到你hexo博客的地址了：

我这里是绑过域名后的，你们看到的和我这个应该不一样，不过你点击访问，就可以访问到你的博客了。
这样，属于你自己的博客就算正式发布上线了，接着我们要去绑定一个域名。
